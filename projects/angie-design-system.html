02

Roles semánticos de color

Cada color primitivo tiene un rol semántico — su intención de uso. Al agregar un componente nuevo, usar el rol (no el valor HEX) garantiza consistencia y facilita cambios globales de paleta en el futuro.

Token semánticoValorMuestraCuándo usarlo
--color-action #F86A43
Botón principal, CTA, enlace de acción primaria
--color-action-bg #FDE8E2
Fondo de tags coral, iconos sobre claro
--color-brand #C2EC9A
Números sobre fondo oscuro (stats), iconos de éxito
--color-brand-dark #5A9E2F
Texto sobre fondos verdes, links de acción secundaria
--color-brand-bg #E8F8D4
Fondo de secciones de transición, hover de cards
--color-surface #FFFFFF
Cards, inputs, nav — cualquier componente elevado
--color-page #FAFAF7
Fondo de página exclusivamente
--color-ink #2C2C2C
Headings, texto cuerpo principal
--color-ink-secondary #5A5A5A
Párrafos, subtítulos, descripciones
--color-ink-tertiary #8A8A8A
Metadatos, placeholders, hints, fechas
--color-line rgba(44,44,44,.10)
Separadores, bordes de sección, hr
--color-line-mid rgba(44,44,44,.15)
Bordes de inputs, cards con más peso visual
Regla de oro: Nunca hardcodear un valor HEX en un componente nuevo. Usar siempre el token semántico. Si necesitas un color que no existe en la tabla, primero agrega el token en angie-tokens.css y luego úsalo.

Escala de Z-Index

Cinco capas nombradas evitan conflictos al agregar modales, tooltips o nuevos componentes fijos. Nunca usar valores arbitrarios como z-index: 9999.

300
--z-toast

200
--z-overlay
Modales, tooltips, popovers, drawers
100
--z-sticky

10
--z-raised

0
--z-base
Flujo normal del documento — no necesita z-index explícito

Tarjetas de servicio — hover interactivo

Tres capas de interacción al pasar el cursor: elevación de la card, animación del icono y aparición del link de acción.

Nutrición Clínica

Evaluación integral y plan personalizado.

/* Acento inferior — reveal en hover */
.servicio-card::after {
  content: ''; position: absolute; bottom: 0; left: 0; right: 0; height: 3px;
  background: linear-gradient(90deg, var(--green-dark), var(--green));
  transform: scaleX(0); transform-origin: left center;
  transition: transform 380ms var(--ease-out);
}
.servicio-card:hover::after { transform: scaleX(1); }

/* Icono — escala y rotación */
.servicio-card:hover .servicio-icon {
  transform: scale(1.12) rotate(-6deg);
  background: var(--green);
}

/* Link "Consultar" — slide in */
.servicio-link { opacity: 0; transform: translateX(-8px); transition: 260ms; }
.servicio-card:hover .servicio-link { opacity: 1; transform: translateX(0); }

Accesibilidad

Checklist de cumplimiento WCAG 2.1 AA — nivel mínimo exigible para cualquier página nueva. Verificar antes de publicar cada sección.

CriterioEspecificaciónEstado
Contraste texto normal ≥ 4.5:1 (texto < 18px) ✓ Cumple
Contraste texto grande ≥ 3:1 (headings ≥ 18px) ✓ Cumple
Focus visible outline 3px coral, offset 2px en todos los interactivos ✓ Cumple
Reduced motion @media (prefers-reduced-motion) elimina todas las animaciones ✓ Cumple
Touch targets Mínimo 44×44px en elementos táctiles ✓ Cumple
Semántica HTML <nav>, <section aria-labelledby>, <article>, <main> ✓ Cumple
Alt text en imágenes Descriptivo y contextual, no decorativo en SVGs con aria-hidden ✓ Cumple
Formulario con labels <label for="id"> explícito en cada campo ✓ Cumple
Errores de formulario aria-live="polite" o role="alert" para mensajes de error Pendiente
Skip to content Enlace invisible que se activa con Tab al inicio de la página Pendiente

Ratios de contraste — pares más usados

#2C2C2C sobre #FAFAF7 17.8:1 ✓
#FFF sobre #F86A43 3.1:1 ⚠
#5A5A5A sobre #FAFAF7 7.4:1 ✓
